# Troubleshooting

## Purpose

Use this page when Unity2Rerun output does not match the expected sample or validation result.

## `.rrd` File Missing Or Empty

Check:

1. `RerunManager` is enabled and recording.
2. Output Mode is `FileOnly` or `FileAndLive`.
3. Output Path points outside the package and to a writable directory.
4. At least one manager API call or publisher emitted data.
5. Play Mode stopped cleanly so the file could close.

Try a short Basic Rrd Recording sample run, then open the Console path with Rerun Viewer.

## Rerun Viewer Cannot Connect

Check:

1. Output Mode is `LiveOnly` or `FileAndLive`.
2. Rerun Viewer is running, or Auto Launch Viewer is enabled.
3. The live endpoint matches the Viewer proxy endpoint.
4. Cysharp `YetAnotherHttpHandler` and its native dependency package are installed.
5. Port `9876` is not owned by an unrelated process.

If live setup is not needed, use `FileOnly` and open the `.rrd` after recording.

## Live Connection Works But No Data Appears

Check:

1. Publishers are enabled and have a positive Publish Rate Hz, or `0` for every frame.
2. A `RerunManager` is present and recording.
3. Entity paths are valid and visible in the Viewer.
4. The Viewer is not showing an old saved layout that filters the current entities.
5. File output contains the expected streams when opened from the saved `.rrd`.

Use `FileAndLive` while diagnosing so the file remains a source of truth.

## RRD Verify Fails

Check:

1. Recording stopped cleanly before verification.
2. The command points to the final `.rrd`, not a file still being written.
3. The file was generated by the current SDK revision.

Run:

```powershell
rerun rrd verify build/RRD/example.rrd
rerun rrd stats build/RRD/example.rrd
```

## How To Verify LZ4 Is Active

File compression is recorded in Rerun ArrowMsg metadata. Do not rely only on file size. Use the compression comparison command in `12_Validation_and_Acceptance.md` and confirm the LZ4 recording reports LZ4-compressed Arrow messages.

Live gRPC payloads remain uncompressed even when file recording compression is LZ4.

## LZ4 File Is Not Smaller Than None

Small recordings, already-compressed images, and metadata-heavy files may not shrink much. LZ4 is active only when the ArrowMsg compression metadata says it is active. Compare stored payload bytes and declared uncompressed bytes rather than only the final file size.

## Camera Image Is Blank Or Missing

Check:

1. `RerunCameraImagePublisher` has an active Camera, or `Camera.main` exists.
2. Width and Height are greater than zero.
3. Format is JPEG or PNG.
4. Max Encoded Bytes is not lower than the encoded frame size.
5. The Unity Game view camera renders visible content.

Oversized dropped frames are logged to `logs/rerun/image`.

## Pinhole And Image Alignment Looks Wrong

Check:

1. `RerunCameraImagePublisher` and `RerunPinholeCameraPublisher` use the same entity path.
2. Width and Height match the captured image resolution.
3. Publish Camera Pose is enabled when you expect a moving camera.
4. ViewCoordinates are written by `RerunManager`.

Use `RepublishPinhole` after changing static pinhole settings during Play Mode.

## Point Cloud Or Laser Scan Is Missing

Check:

1. Point cloud sources are assigned, or Use Children When Sources Empty is enabled and child transforms exist.
2. Laser scan range limits allow the generated or external ranges.
3. Publish Points and/or Publish Line Strip are enabled.
4. Entity paths match what you are viewing in Rerun.

The laser scan outline is emitted at the scan entity path with `_outline` appended.

## Generated `[RerunLog]` Data Is Missing

Check:

1. The annotated class is `partial`.
2. The scene has an active `RerunManager`.
3. The generated source component is active in Play Mode.
4. RateHz values are not so low that the sample ends before a publish tick.

The Generated RerunLog sample should emit `logs/generated`, `metrics/generated_fps`, and `world/generated_cube`.

## Sidecar Control URL Does Not Open

Check:

1. `RerunInteractiveControlBridge` is enabled.
2. Start On Enable is enabled, or `StartServer()` was called.
3. Preferred Port is free, or read the runtime Control URL field for the fallback port.
4. Unity keeps updating while the browser has focus; keep Run In Background enabled.

The sidecar is local loopback only. It is intended for sample control, not remote production control.

## IL2CPP Build Or Player Runtime Issue

Check:

1. The Unity target platform module is installed.
2. The package `Runtime/link.xml` is included.
3. Required plugin DLLs are present under `Runtime/Plugins`.
4. Live gRPC dependencies are installed in the consuming project when live output is used.
5. FileOnly `.rrd` recording works before diagnosing live output.

Use the IL2CPP build guide for the complete build workflow.
